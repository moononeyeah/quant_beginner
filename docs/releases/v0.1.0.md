# v0.1.0 - Initial Open Source Release

`quant_beginner` is now available as an open source Python project for learning A-share and ETF quantitative backtesting.

This release focuses on a maintainable backtesting foundation, a usable local UI, and a set of vn.py-style preset strategies that can be studied and extended.

## Highlights

- Visual Gradio backtesting workbench.
- CTA-style backtesting engine inspired by vn.py.
- ETF rotation portfolio backtest.
- Strategy registry with parameter metadata, source preview and dynamic strategy selection.
- Parameter optimization from both CLI and UI.
- Daily mark-to-market accounting with commission, slippage, equity curve, drawdown, Sharpe ratio and win-rate statistics.
- Data fetching through public market data interfaces with local cache fallback.

## Quick start

```bash
git clone https://github.com/moononeyeah/quant_beginner.git
cd quant_beginner
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Then open the local Gradio URL shown in the terminal.

## CLI examples

Run a double moving average backtest:

```bash
python main.py --strategy double_ma --symbol 510300 --frequency daily --start 20230101 --end 20260508
```

Run ETF rotation:

```bash
python main.py --strategy rotation --symbol 510300 --rotation-symbols 510300,159915,512100,518880,513100 --frequency daily --start 20230101 --end 20260508
```

Run parameter optimization:

```bash
python main.py --strategy double_ma --symbol 510300 --frequency daily --start 20230101 --end 20260508 --optimize '{"fast_window":[5,10,20],"slow_window":[20,30,60]}'
```

## Repository hygiene

This release intentionally excludes:

- `.venv/`
- `data/`
- `outputs/`
- `.cache/`
- Python bytecode and local generated files

No credentials, private configuration, local market cache or generated chart output are included.

## Known limitations

- This project is for learning and research only. It is not a live trading system.
- Public market data APIs may fail because of network, proxy or upstream changes.
- Minute data usually covers only recent periods.
- The engine does not currently model limit-up/limit-down execution failure, suspensions, order book depth, tax, dividends or real brokerage constraints.
- The alpha/cross-sectional vn.py demo strategy is not included because it needs a separate alpha engine and signal data model.
