# v0.3.0 Release Notes

## Highlights

This release focuses on three themes:

1. **Stability**: data fetch resilience with retry/backoff and cache fallback.
2. **Reproducibility**: complete run context persistence and one-click history rerun.
3. **Workflow UX**: optimization profile save/load and reusable best-parameter flow.

## What's New

### Data reliability

- Added retry + exponential backoff in `src/data_fetcher.py`.
- Added data fetch health logging into `data/data_health.db`.
- Added `load_data_health()` API for recent health records.
- Added index daily compatibility for common index symbols (including `000300`).

### Frontend workflow

- Added sidebar data health panel in `app.py`.
- Added one-click rerun by record ID in `pages/5_历史记录.py`.
- Added optimization profile save/load in `pages/3_参数优化.py`.
- Added drawdown-threshold filtering in optimization results.
- Added "save best params to session" and auto-prefill in single/portfolio backtest pages.

### Tooling and docs

- Added `examples/update_test_data.py` for bulk cache refresh.
- Updated README for new workflow and resilience capabilities.

## Notes

- Data pull success still depends on upstream data providers and local network/proxy state.
- This project remains a learning/research platform and is not intended for live trading.
